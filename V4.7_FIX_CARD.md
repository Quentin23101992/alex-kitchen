# Alex's Kitchen — v4.7 Fix Card
**Date:** 2 July 2026 | **Base:** v4.6 (4803 lines) | **Type:** bug fixes only, zero new features (Phase 1 compliant — one exception noted below)

## P1 — Fixed in the patched file

### 1. UTC date bug (shift-breaking after 22:00 Paris time)
`S.d = new Date().toISOString().split('T')[0]` returns the **UTC** date. Paris is UTC+2 in summer.
**Failure scenario:** Alex closes a PM shift, iOS kills the PWA in background (it does this within minutes), she reopens at 22:30 → app initializes with *tomorrow's* date → her whole shift appears empty and the Opening Ritual overlay pops up while she's trying to close. Same bug silently killed prep deadline colors after 22:00 (`prepStatus` compares against UTC today).
**Fix:** added `todayLocal()` helper; replaced init, `prepStatus`, `checkBackupPrompt`, and backup filenames (5 call sites).
**Not fixed (needs a decision, see below):** true after-midnight shifts (00:00–02:00 local) will still land on the next day's date even with local time.

### 2. Tell Q had no read path — Phase 1 instrument was write-only
`saveTellQ()` stores answers into `S.tellQ` + `km3_tellq`, but **nothing in the app reads either one**. Alex's answers to THE Phase 1 question were being saved into a hole: no viewer, and `km3_tellq` was excluded from backup AND restore.
**Fix:**
- `km3_tellq` now included in `backupJSON()` + `doAutoBackup()` as `_tellq`, restored by `importJSON()`.
- New button in Settings & Data: **"Copy 'Tell Q' notes"** → aggregates all entries (log + per-shift, deduped by timestamp), copies as `date shift — text` lines. Alex taps once, pastes into WhatsApp to you. *(This is the one "new" UI element — it's the retrieval half of an existing Phase 1 feature, not a new feature.)*

## P2 — Fixed

### 3. Dishonest storage toast
At 90% storage, `lsQuotaCheck` toasted "old shifts archived" **before** the user answered the delete-confirm — shown even if she pressed Cancel, and "archived" actually means *deleted*. Reworded to "Storage 90% full"; `archiveOldShifts()` already reports honestly after consent.

### 4. Silent-failing Defer button
`deferFollowUp()` returned silently if `S.deferredItems` was undefined (true on fresh shifts created before v4.3-era fields existed). Now initializes the array instead.
